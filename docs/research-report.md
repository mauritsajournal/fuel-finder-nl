## 1. Executive Summary

Fuel Finder NL is a proof-of-concept web application that shows users the cheapest fuel stations in the Netherlands on an interactive map. The app will also display EV charging stations (with real-time status and pricing), nearby facilities (playgrounds, toilets, rest areas), and calculate actual driving time/distance to selected stations. The PoC is a stepping stone toward a production-quality mobile app covering all of Europe.

Research reveals that **per-station fuel price data in the Netherlands is obtainable** but not through a single clean open API. The two viable sources are the ANWB Onderweg API (public, no auth, JSON) and the DirectLease TankService API (requires checksum generation). Both have been reverse-engineered by the Home Assistant community and documented in the `pyfuelprices` Python library. For EV charging, OpenChargeMap provides a free global API with location data, connector types, and some pricing -- but real-time occupancy requires the Dutch government's DOT-NL platform (requires registration). For the PoC, OpenChargeMap is sufficient.

The critical architectural insight is that **fuel price APIs cannot be called from the browser** (CORS restrictions, checksum requirements, rate limits). Following the same pattern as the FPL Draft Dashboard, we will use **GitHub Actions as a server-side data pipeline**: a scheduled workflow fetches fuel prices and processes them. However, unlike the FPL project, the data volume here is too large to commit to git every 4 hours (4000+ stations + 30k chargers would bloat the repo within months). Instead, processed data will be stored in **Cloudflare R2** (free tier: 10GB storage, 10M reads/month) and served as static JSON files via a public bucket URL. The frontend is a pure static SvelteKit site on GitHub Pages that fetches data from R2.

**Important legal note:** The ANWB and DirectLease APIs are undocumented/reverse-engineered. Using them in a public product carries legal risk under EU database rights (Directive 96/9/EC). For the PoC, this is acceptable as a technical demonstration. For a production product, formal data partnerships or official API access agreements would be required. This risk is explicitly documented in Section 6.

### 2.3 Constraints & Assumptions

- **Minimal server-side infrastructure.** Data processing happens in GitHub Actions. Data storage in Cloudflare R2 (free tier). The deployed site on GitHub Pages is 100% static files.
- **API rate limits apply.** ANWB and DirectLease APIs should only be called from GitHub Actions (server-side), not from the browser.
- **Data staleness is acceptable for PoC.** Prices updated every 4-6 hours is fine. Real-time per-second pricing is not expected.
- **Netherlands only for PoC.** Architecture should allow adding countries later, but PoC scope is NL.
- **OpenRouteService requires a free API key.** Routing requests will be proxied through a Cloudflare Worker (free tier: 100k requests/day) to protect the API key from abuse.
- **OpenChargeMap API key required for >250 results.** Free registration needed.
- **Assumption:** ANWB API remains publicly accessible without authentication (as observed March 2026). If it changes, DirectLease is the fallback.
- **Legal constraint:** Both ANWB and DirectLease APIs are undocumented. Using them publicly carries legal risk. PoC is acceptable; production would need formal agreements.
- **Data loading must be viewport-based.** Loading 10MB+ of JSON on mobile is not acceptable. Data will be split into geographic tiles and loaded on demand.

### 3.3 Key Insights

1. **No competitor covers NL + EV + facilities + routing in one free app.** ANWB Onderweg comes closest but lacks real-time EV status and modern UX. This is a genuine gap.

2. **NL fuel price data is available but fragmented.** The ANWB API (public, no auth, JSON with prices) is the cleanest source. DirectLease has more data but requires checksum authentication. Both are documented in `pyfuelprices`.

3. **Nobody does "cheapest fuel + EV charging + facilities" in one view.** This is the unique value proposition. Competitors are either fuel-only or EV-only.

4. **Route-based search is a premium feature everywhere.** 1-2-3 Fuel charges for it. If we can offer it free (via OpenRouteService), that is a differentiator.

5. **Modern UI is a real differentiator.** Most fuel apps look dated. Fillzz is the exception with excellent design. A glassmorphism dark theme would stand out.

6. **Europe expansion is feasible.** Germany has TankerKoenig (free, official, excellent). France, Spain, Italy have official government data. The `pyfuelprices` library already supports 14 countries.

## 7. Recommendations

### Start with ANWB Only, Drop DirectLease from PoC

The ANWB API is simpler (GET request, no auth, JSON response with prices) and provides sufficient coverage (~3,800 stations) for a convincing PoC. DirectLease adds complexity (checksum auth, fragile reverse-engineering) without proportional value for a demo. Add DirectLease only if ANWB proves insufficient or as a Phase 2+ enhancement.

### Store Data in Cloudflare R2, Not Git

Do not commit multi-MB JSON files to the git repo every 4 hours. This bloats the repo quickly and is an anti-pattern. Use Cloudflare R2 (free tier: 10GB) for data storage. The GitHub Actions pipeline uploads processed JSON to R2, and the frontend fetches from the R2 public URL. Git stays clean for code only.

### Proxy the Routing API Through Cloudflare Workers

Never expose API keys in client-side JavaScript. Use a Cloudflare Worker (free tier: 100k requests/day) as a proxy for OpenRouteService. The Worker validates requests, adds the API key server-side, and forwards to ORS. This prevents key abuse and allows rate limiting at the proxy level.

### Use MapLibre GL JS, Not Leaflet

For a modern, app-like feel with smooth zooming and hundreds of markers, MapLibre with vector tiles is the right choice. The glassmorphism dark theme also pairs better with vector map styling (you can style the map itself to be dark).

### Use Viewport-Based Data Loading

Do not load 10MB+ of JSON data on page load. Split data into geographic tiles (e.g., by province or grid cells). Load only the tiles visible in the current map viewport. Use MapLibre's `moveend` event to fetch additional tiles as the user pans. This keeps initial load under 500KB and scales well for European expansion.

### Invest in UI -- This Is a Product Pitch

Since the goal is to eventually build a real product, the PoC needs to look convincing. Spend time on the glassmorphism dark theme, smooth map interactions, and a clean station detail panel. First impressions matter for investor/partner conversations.

### Plan the Data Architecture for Europe

Even though the PoC is NL-only, structure the data pipeline so adding new countries is straightforward. Use a country-based folder structure for JSON data files. The `pyfuelprices` library already supports 14 countries, so the expansion path is clear.

### Consider PWA from Day One

A SvelteKit static site can be made PWA-capable with minimal effort (service worker + manifest). This provides "install to home screen" functionality on mobile, which bridges the gap to a native app without the development cost. For the PoC, this is a quick win that makes it feel more like a real app.
